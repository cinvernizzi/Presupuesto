Relevamiento de la estructura de datos de una aplicación de presupuesto

Tener en cuenta que SQLite si maneja triggers, vistas e índices foráneos
pero los tipos de datos son mas limitados

Necesitamos una tabla donde figuren los datos nuestros como mínimo
1. El logo que utilizaremos
2. El nombre de la Empresa
3. La dirección de la misma
4. CUIT / CUIL datos de la misma
5. Teléfono 
6. Mail 
7. Posiblemente firma para la presentación de los presupuestos

Vamos a tener una tabla de Secciones que contendrá básicamente (cada 
registro debería contener información sobre, el orden de presentación 
y si se debe incluir o no en el presupuesto para el cliente)
1. Análisis y Diseño
2. Desarrollo Backend
3. Desarrollo FrontEnd
4. Testing
5. Deploy y Configuración
6. Hosting y Licencias

Vamos a necesitar otra tabla de clientes que contendrá
1. Nombre del Cliente
2. Dirección Postal
3. Identificación de impuestos (CUIT)
4. Teléfono
5. Mail 

La tabla de presupuestos contendrá 
1. Id de presupuesto
2. Fecha del presupuesto
3. Validez en días
4. Id del proyecto

A su vez tendremos una tabla de proyectos (que será la madre)
1. Clave del proyecto
2. Título del proyecto
3. Definición ampliada del mismo
4. Configuración del proyecto (si se incluyen costos de gestión 
como item independiente o si se calcula un porcentaje y en 
tal caso, el valor de ese porcentaje)

A su vez vamos a necesitar una tabla de items (tareas) del proyecto
1. Clave del proyecto
2. Clave de la sección (el diccionario)
3. Descripción de la actividad
4. Horas estimadas
5. Costo unitario (por hora)
5. Consideraciones

El sistema debería entonces ordenar las tareas de acuerdo al orden 
de las mismas y calcular el costo total de cada actividad y el 
parcial por cada una de las etapas

Luego debería generar un pie con el total de cada una de las 
etapas y el costo final
